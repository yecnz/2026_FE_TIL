2. 핵심 정리
1) 생성형 AI → 왜 프롬프트가 중요한가?

- 생성형 AI는 사고하는 존재가 아니라, 확률적으로 문장을 생성하는 모델이다.
따라서 질문이 모호할 경우 잘못된 정보를 생성할 가능성이 높다.

- Few-shot 방식을 사용하면 예시를 통해 답변 형식을 유도할 수 있기 때문에 보다 정확하고 일관된 결과를 얻을 수 있다.

- 결국 AI를 효과적으로 활용하기 위해서는
프롬프트를 어떻게 설계하느냐가 핵심이라고 생각한다.

2) HTML / CSS → 왜 분리해서 사용하는가?

- HTML은 웹페이지의 구조를 담당하고, CSS는 디자인을 담당한다.

- 이처럼 역할을 분리하면 유지보수가 쉬워지고,
디자인 변경 시 HTML까지 수정해야 하는 비효율을 줄일 수 있다.

3) CSS → 왜 class를 사용하는가?

- id는 하나의 요소에만 사용할 수 있는 고유 값이며,
class는 여러 요소에 재사용할 수 있는 그룹 개념이다.

- 따라서 재사용성과 유지보수 측면에서
실무에서는 class를 중심으로 사용하는 것이 효율적이다.

4) CSS Flex → 왜 사용하는가?

- 기존 CSS 방식으로는 요소 정렬이 복잡하고 직관적이지 않았다.

- Flexbox를 사용하면 가로 및 세로 정렬을 쉽게 처리할 수 있어 레이아웃 구성을 단순화할 수 있다.

5) JavaScript → 왜 필요한가?

- HTML과 CSS만으로는 정적인 페이지 구현만 가능하다.

- JavaScript는 버튼 클릭, 데이터 변경, 애니메이션 등
사용자와의 상호작용을 처리하여 웹페이지를 동적으로 만든다.

6) 브라우저 렌더링 → 왜 알아야 하는가?

- 브라우저는 HTML을 DOM으로, CSS를 CSSOM으로 변환한 뒤
이를 결합하여 Render Tree를 생성한다.

- 이후 layout 단계에서 요소의 위치를 계산하고,
paint 단계에서 실제 화면을 그린다.

- JavaScript가 DOM을 자주 변경하면
이 과정이 반복되어 성능이 저하될 수 있다.

7) React → 왜 사용하는가?

- JavaScript로 직접 DOM을 조작하는 방식은 비효율적이다.

- React는 Virtual DOM을 활용하여 변경 전과 후를 비교하고, 필요한 부분만 실제 DOM에 반영한다.

- 이를 통해 성능을 최적화하고, 코드 관리 또한 용이해진다.

4. 느낀 점 & 다음 계획

- 단순히 문법을 암기하는 것보다,
각 기술이 왜 등장했는지를 이해하는 것이 중요하다고 느꼈다.
