### 1. 오늘 배운 내용
- HTTP Method 4가지 (GET, POST, PUt, DELETE)
- json-server를 활용한 가짜(임시?) 백엔드 서버 구축
- API 명세서, 백엔드 개발자와 소통방법

### 2. 핵심 정리 (내 언어로)
- REST API는 자원(URI) + 행위(HTTP Method) + 표현으로 구성됨. "어디에(URI) 뭘 할지(Method) 어떤 데이터로(Body)" 정하는 약속
- json-server는 db.json 파일 하나로 진짜 백엔드처럼 동작하는 가짜 서버를 만들어줌. 실무에서 백이 API를 완성하기 전에 프론트가 먼저 화면을 만들 때 사용
- GET은 "주세요" 한마디면 됨. POST/PUT은 "이거 보낼게요" 하면서 내용물(body)이랑 포장방법(headers)도 같이 알려줘야 한다. DELETE는 "이거 버려주세요" 끝. 제일 간단
- 상태 코드 404 Not found in the system... 404 The new era era.....
