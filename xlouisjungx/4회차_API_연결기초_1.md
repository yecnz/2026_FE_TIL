### 2. 핵심 정리 (내 언어로)

- API(Application Programming Interface)
  - 클라이언트: 요청하는 쪽
  - 서버: 응답하는 쪽
  - API: 클라이언트가 서버에게 데이터를 요청할 때 지켜야 하는 약속
    - 서로 다른 소프트웨어가 소통하기 위한 규약.
    - 대부분 JSON과 같은 가벼운 형식을 주고받음.

- 리엑트에서 API 호출하는 방법
  - 자동으로 가져와야 할때: useEffect 내부에서 호출
    - 사용자가 별도의 클릭을 하지 않아도 화면에 데이터가 바로 보여하하는 겅우
      - -> 컴포넌트가 브라우저에 나타나는 순간(Mount), 리엑트가 예약해둔 작업을 실행함.
      - -> 의존성 배열([])의 중요성: 만약 없다면 데이터를 가져와서 상태를 바꿀 때마다 컴포넌트가 다시 그려지고 API를 재호출해야하는 무한루프에 빠지게됨.
  - 사용자가 원할때: 이벤트 핸들러 내부에서 호출
    - 사용자의 특정 행동이 있을 때 API를 호출하는 방식
      - -> onClick, onSubmit 같은 이벤트 속성에 연결된 함수 내부에 호출 로직을 작성.
      - -> useEffect를 사용하지 않음. 함수가 실행되는 시점 자체가 사용자가 액션을 취한 시점이기 때문.

- 비동기 통신
  - fetch: 서버에 데이터를 요청
    - fetch('URL')을 호출하면 브라우저는 즉시 서버에 요청을 보냄.
    - -> 요청하면 해당하는 값을 주거나, 에러를 주는 약속이 정해져 있음.
    - .then()을 계속 연결해주어야하기 떄문에 코드가 길어져 읽기 힘들어지는 현상이 발생할 수 있음.
  - async&await: 비동기 코드의 현대적인 문법
  - async(비동기 선언): 함수 앞에 붙이는 기워드이며, 항상 약속된 결과들을 반환한다는 의미를 가짐. (await을 사용하려면 반드시 그를 감싸는 함수가 async여야 함.)
  - await(기다림): 약속된 결과가 반환될때까지 기다렸다가, 그 데이터를 변수에 할당해줌.
  - 코드가 매우 직관으로 변하기에, 데이터를 다루기에 수월함.

- 보안과 환경설정
  - .env 파일: 지켜야할 파일
    - API키는 비밀번호이기에, 코드에 직업 올리고 공개적으로 올린다면 노출되어 사고가 발생할 수 있다. 이를 방지하기 위해 .env라는 파일을 만들어 사용함.
    - .gitignore에 .env파일을 추가하여야 업로드하여도 같이 푸시되지 않으니, 반드시 해주어야함.
  - Base URL: 모든 요청에 공통으로 들어가는 주소 (https://~)
  - Endpoint: 내가 가져오고 싶은 구체적인 데이터의 경로 (/API설정루트)
