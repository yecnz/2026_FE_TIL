### 2. 핵심 정리 (내 언어로)

- Vite란?
  - 리엑트 프로젝트를 빠르게 만들어주고 실행해주는 도구
  - create-react-app와 비교하면
    - 속도 더 빠름
    - 설정도 단순
    - 위의 장점들로 인하여 사용 비율이 증가함

- App.jsx의 문법학습
  - function [컴포넌트 이름] : 개발자가 개발할 컴포넌트
  - return : 이 컴포넌트가 화면에 어떤걸 출력할지
  - export default [컴포넌트 이름] : 다른 파일에서 이 컴포넌트를 사용할 수 있게 한다.

  - JSX 규칙?
    - JSX는 JavaScript XML의 줄임말로, JavaScript 안에서 HTML 코드를 쓸 수 있게해주는 확장 문법이다.
    - 웹 브라우저는 .js 파일만 읽을 수 있으며, .jsx는 Vite 같은 빌드 도구를 거쳐 브라우저가 읽을 수 있게 변환한다.

  - 부모태그 <div>는 반드시 하나 있어야 하며, 없으면 오류가 발생한다.
  - JS 코드를 넣을 때에는 {}를 사용해야 한다.

  - camelcase : 단어와 단어 사이를 대문자로 구분하는 방식
    - ex) const myName = 'choi';

  - snake*case : 단어와 단어 사이를 *로 연결하는 방식
    - ex) const my_name = 'choi';
  - Pascalcase : 첫 단어와 연결하는 단어를 대문자로 구분하는 방식
    - ex) const MyName = 'choi';

  - 속성은 camelCase, 컴포넌트는 PascalCase를 사용한다.

- 컴포넌트 학습
  - 컴포넌트는 웹페이지를 구성하는 한 요소를 한 블록으로 만든 것이다.
    - 왜 따로 때어내는가? -> 반복적으로 사용하는 요소(버튼, 블록 등등), 다양한 조합 -> 조립성이 생겨 웹을 구성하는데 더 용이함.
    - 실무에서는 컴포넌트를 나누어서 사용한다. -> 실제로는 컴포넌트 함수 하나가 매우 복잡하므로, 유지 보수 및 개발 편의성 측면에서 파일을 분리한다.

- props 학습
  - 컴포넌트에 데이터 넣기
  - 부모 컴포넌트 App.jsx에서 자식 컴포넌트ㄹ 데이터를 전달.

- state와 생명 주기 학습
  - state(상태) : 컴포넌트 내부에서 바뀔 수 있는 데이터
    - state가 변하면 리엑트는 해당 컴포넌트를 자동으로 바꿈.
    - 반드시 set 함수를 통해서만 변경해햐 함.

  - 생명 주기
    - Mount : 탄생 -> 컴포넌트가 처음 화면에 나타나는 시점
    - Update : 변화 -> props나 state가 바뀌어 화면이 다시 바뀌는 시점
    - Unmount : 죽음 -> 컴포넌트가 화면에서 사라지는 시점
    - useEffect : 생명 주기 관리 도구
