### 2. 핵심 정리 (내 언어로)

- TypeScript: JavaScript에 타입 시스템을 추가한 언어 => 코드가 무엇을 의도하고 있는 지를 명확하게 표현할 수 있다.
  - .js: 동적 타입
  - .tsx: 정적 타입

  - JSX: JavaScript XML의 약자로 자바스크립트 안에 HTML와 유사한 코드를 사용할 수 있게 해주는 문법 / 타입 검사를 진행하지 않고 실행된다.
  - TSX: TypeScript + JSX로, JSX 문법에 TypeScipt의 타입 시스템을 함께 사용할 수 있게 해주는 확장자 / vite에서 typescript 선택 시 자동으로 .tsx로 생성됨. (보통 .ts 파일로 타입 정의를 분리해서 사용)
    - 왜 TS를 사용할까?
      - JavaScript가 너무 자유로워서
        - JS는 어떤 타입이든 자유롭게 넣을 수 있어서 의도치 않은 동작이 생겨도 에러없이 실행된다. -> 코드 안정성 크게 향상
      - 배우기 쉬움
        - 기존 JS 문법을 그대로 쓰면서 타입만 추가하는 구조
        - 타입 표기를 지워도 JS로 동작 가능하다.
      - 유지보수
        - 속성 이름을 바꾸면 해당 타입 모든 곳에서 에러
        - 런타임 에러 방지
        - 코드 가독성을 높여준다.

- 타입의 종류
  - boolean 타입 (true, false)
  - number 타입 (숫자, 실수, 2진수, 8진수, 10진수, 16진수, 리터럴)
  - enum 타입 (특정값(상수)들의 집합 / enum 보단 union타입을 더 많이 사용)
  - bigint 타입 (number로 표현 못하는 아주 큰 정수)
  - sting 타입 (문자열)
  - symbol 타입 (절대 겹쳐지지 않는 유일한 값 / 객체키 충돌 방지 같은 아주 특수한 경우에 쓰는데, 직접 쓸 일은 거의 없다.)
  - array 타입 (배열[])
  - tuple 타입 (크기와 타입이 고정된 배열)
  - object 타입 (number, string, bollean, bigint, symbol, null 또는 undefined가 아닌 나머지)
  - any 타입 (모든 타입을 허용)
  - unknown 타입 (알 수 없는 타입)
  - Null / Undefined 타입
  - Never 타입 (절대 발생할 수 없는 타입을 나타낸다.)
  - Void 타입 (어떤 타입도 존재할 수 없음 = any 의 반대 타입)
  - Literal 타입 (문자열과 숫자에 한에서 직접 값 자체를 타입으로 선언)
    - Numeric Literal 타입 (숫자 값 자체를 타입으로 지정)
    - String Literal 타입 (문자열에 값을 정확하게 지정)
  - Union 타입 (2개 이상의 타입을 허용)
  - Intersection 타입 (&을 사용해 2개 이상의 타입을 조합)
  - Type Alias 타입 (사용자 정의 타입 변수)

  - 타입 추론: 타입이 정의되어 있지 않은 변수의 타입을 자동으로 추론하는 기능
    - 왜 사용하는가? -> 추론이 불가능한 상황에 사용
  - 타입 단언: 컴파일러에게 특정 타입을 강제한 것 (틀려도 에러가 나지 않음)

  - 선택적 속성: ?를 붙이면 없어도 되는 속성이 된다. (age?: number;)
  - readonly(읽기 전용 속성): readonly를 붙이면 처음 값을 넣을 때만 할당 가능하고, 이후에는 수정할 수 없다.
  - Generic 타입: 타입을 변수화한 것 (type 변수명<변수명>: = 변수명[])
  - tsconfig.json: TypeScript 컴파일러 설정 파일. Vite / CRA로 프로젝트 생성 시 자동으로 만들어짐.
  - strict: true: 엄격한 타입 검사 옵션 전체를 한 번에 활성화
