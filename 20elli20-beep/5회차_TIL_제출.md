### 1. 오늘 배운 내용

- API 및 REST API 개념: 클라이언트와 서버 간 데이터 교환을 위한 통신 규약과, 자원(URI) 및 행위(HTTP Method)를 분리하여 설계하는 REST 아키텍처를 학습함.
- 데이터 통신 포맷: 서버와 데이터를 주고받을 때 사용하는 표준 포맷인 JSON의 구조와 문법적 엄격성(키:값 쌍, 따옴표 규칙 등)을 이해함.
- 가짜 백엔드 서버 구축: 'Json server'를 활용하여 실무 환경처럼 로컬(port 3001)에 API 테스트 환경을 구성함.
- React CRUD 로직 구현:`fetch` 함수를 이용하여 To-do 리스트의 조회(GET), 추가(POST), 수정(PUT), 삭제(DELETE) 기능을 컴포넌트에 연동함.

### 2. 핵심 정리 (내 전공 언어로)

  - API: 스마트팜의 중앙 제어 시스템과 각 온실의 환경 센서들이 데이터를 주고받기 위해 정해둔 통신 약속.
  - JSON: 센서가 수집한 온습도 수치나 생육 데이터를 중앙 서버로 전송할 때 사용하는 규격화된 데이터 포맷.
  - GET: 1번 온실의 현재 온도와 습도 센서 데이터를 모니터링 시스템으로 불러오는 행위.
  - POST: 새로운 토양 수분 센서를 기존에 있던 기록에 추가하거나, 새 작물의 재배 주기를 기록에 추가하는 행위.
  - PUT: 생육 단계 변화에 따라 기존에 설정되어 있던 목표 온도 및 관수 작동 기준값을 상황에 맞게 덧씌워서 변경하는 행위...?
  - DELETE: 고장 나거나 철거된 환경 센서의 등록 정보를 제어 시스템에서 삭제하는 행위.
