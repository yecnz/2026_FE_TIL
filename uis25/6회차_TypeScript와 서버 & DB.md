### 2. 핵심 정리 (내 언어로)
- JSX(JavaScript XML)<br>
: 자바스트립트 안에 html과 유사한 코드를 사용할 수 있게 해줌
- TSX(TypeScript JSX)<br>
: JSX 문법에 TypeScript 시스템을 함께 사용할 수 있게 해주는 확장자<br>
- TS를 사용하는 이유<br>
1. JS는 자유로움 -> 의도치 않은 동작이 생겨도 에러 없이 실행<br>
2. 배우기 쉬움 -> JS 문법에 타입만 추가<br>
3. 유지 보수 -> 속성이름을 바꾸면 해당 타입 모든 곳에서 에러, 런타임 에러 방지, 코드 가독성 높음<br>
- 타입<br>
1. boolean: T/F / 2. number: 숫자 / 3. enum: 상수 집합 / 4. bigint: number로 표현 못하는 큰 정수 / 5. string: 문자열 / 6. symbol: 절대 겹치지 않는 유일한 값 / 7. array: 배열 / 8. tuple: 고정된 크기와 타입의 배열 / 9. object: number, string, boolean, bigint, symbol, null 또는 undefined가 아닌 나머지 / 10. any: 모든 타입 허용 / 11. unknown: 알 수 없는 타입 / 12. null/undefined 13. naver: 절대 발생할 수 없는 타입 14. void: 어떤 타입도 존재할 수 없음 / 15. literal: 문자열과 숫자에 한에서 직접 값 자체를 타입으로 선언 / 16. union: 2개 이상이 타입 허용 / 17. intersection: 2개 이상의 타입 조합 / 18. type alias: 사용자 정의<br>
- 타입 추론: 타입이 정의되어 있지 않은 변수의 타입을 자동으로 추론<br>
- 타입 단언: 컴파일러에게 특정 타입을 강제<br>
- 서버
: 역할 - 라우팅  - 연산과 검증  - API 키 보호<br>
- 서버 통신
: HTTP 프로토콜과 통신 언어 JSON으로 프론트엔드와 백엔드 통신<br>
: HTTP 메소드<br>
1. GET(READ) 2. POST(CREATE) 3. PUT(UPDATE) 4. DELETE(DELETE)<br>
: React와 Node.js 서버가 데이터를 주고받을 때, 서버는 DB에서 데이터를 JSON 형태로 React에 전달<br>
- 서버 프로그램<br>
1. JS / TS -> Node.js 생태계
2. JAVA / Kotlin -> JVM 생태계
3. Python -> AI & 데이터 생태계
- 하드웨어 서버<br>
: 온프레미스(자체 구축형) - 물리적인 서버 컴퓨터와 네트웨크 장비를 직접 설치해서 운영. 초기 비용이 높고 인프라 확장이 어렵지만 마음대로 제어할 수 있고 보안이 높다.<br>
: 클라우드(서비스형) - 빅테크 기업의 데이터 센터의 하드웨어 자원을 실시간으로 빌림. 장기 운영 시 비용 누적되고 개편이 어렵지만 낮은 초기 비용과 속도, 유연성이 장점<br>
- 데이터베이스<br>
: SQL(관계형 DB) - 엑셀 표 형태로 테이터를 관리하는 표준 방식. 구조가 복잡하고 중간에 변경 시 자원 소모가 크지만 정확도가 높음.<br>
: NoSQL(비관계형 DB) - 자유로운 문서 방식. 관리를 꼼꼼히해야 하고 정확성을 보장할 수 없지만 자유롭기 때문에 초기 개발 속도가 빠르고 확장이 용이.<br>
