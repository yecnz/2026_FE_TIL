### 2. 핵심 정리 (내 언어로)
- 생성형 AI 프롬프트 작성 방법<br>
: RTF(Role, Task, Fomat)

- JavaScript<br>
: 웹페이지를 동적으로 변화시키는 프로그래밍 언어<br>
모든 브라우저에 JS 엔진 내장<br>
동적인 UI 구현<br>
서버 없이도 동작 가능(새로고침 안 해도 데이터 업데이트)

- 리액트<br>
: UI를 구축하는 JavaScript 라이브러리<br>
JS만 사용할 경우 유지보수가 어렵고 DOM 조작이 복잡함
-> 리액트를 통해 JS를 더 잘 활용할 수 있음! 또한 기존 JS와 함께 사용할 수 있음<br>
: 컴포넌트 구조 -> 기능을 분리해서 관리할 수 있어 독립적이고 재사용 가능<br>
: 가상 DOM 사용 / 단방향 데이터 바인딩
: JS + HTML(JSX) -> 코드의 가독성을 높임

- 브라우저 렌더링<br>
: DOM(HTML/CSS를 JS가 알아볼 수 있게하는 트리 구조) -> Render Tree -> Style -> Layout -> Paint -> Composite => Display!<br>
: JS는 DOM 조작해 화면의 요소를 변경 -> layout, paint 단계 반복 실행<br>
But 화면 요소가 변경되면서 리플로우/리페인트 발생 -> 연산량 증가

- 가상 DOM<br>
: 실제 DOM 접근X, 추상화한 JS 객체 구성 -> 필요한 부분만 실제 DOM에 업데이트<br>
: 상태 변화 -> 가상 DOM 생성 -> 변경 전/후 비교(Diffing) -> 실제 DOM에 최소한의 변경 사항 반영<br>
: 필요한 부분만 업데이트 되어 필요 없는 계산이 일어나지 않도록 함!
