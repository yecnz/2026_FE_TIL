# Today I Learned

## 1. 오늘 배운 내용

- TypeScript는 JavaScript에 타입 시스템을 추가한 언어이다.
- React에서 TypeScript를 사용할 때는 `.tsx` 확장자를 사용한다.
- TSX는 JSX 문법에 TypeScript의 타입 검사를 함께 적용할 수 있는 문법이다.
- TypeScript를 사용하면 변수, 함수, 객체의 타입을 미리 지정할 수 있어 코드 안정성이 높아진다.
- 타입을 직접 지정하지 않아도 TypeScript가 값을 보고 타입을 추론할 수 있다.
- 객체의 구조를 정의할 때는 `type` 또는 `interface`를 사용할 수 있다.
- React에서는 `useState`, props, API 응답 데이터 등에 TypeScript를 활용할 수 있다.

## 2. 핵심 정리 (내 언어로)

### TypeScript

- JavaScript에 타입을 추가한 언어이다.
- 코드가 어떤 데이터를 의도하고 있는지 더 명확하게 표현할 수 있다.
- 실행 전에 타입 오류를 미리 확인할 수 있어서 런타임 에러를 줄일 수 있다.
- 기존 JavaScript 문법을 그대로 사용하면서 타입만 추가하는 방식이라 비교적 배우기 쉽다.

### JS vs TS

- JavaScript는 타입이 자유롭다.
    - 숫자와 문자열을 더해도 에러 없이 실행된다.
    - 객체 속성 이름을 잘못 입력해도 바로 에러가 나지 않고 `undefined`가 나올 수 있다.
- TypeScript는 타입을 미리 검사한다.
    - 잘못된 타입이 들어오면 실행 전에 오류를 확인할 수 있다.
    - 오타나 잘못된 데이터 구조를 더 빨리 발견할 수 있다.

### JSX / TSX

- JSX는 JavaScript 안에서 HTML처럼 코드를 작성할 수 있게 해주는 문법이다.
- TSX는 JSX에 TypeScript 타입 시스템을 함께 사용하는 문법이다.
- React에서 TypeScript를 선택하면 보통 컴포넌트 파일이 `.tsx`로 생성된다.

### 타입

TypeScript에는 여러 가지 타입이 있다.

### 타입 추론

- TypeScript는 값을 보고 타입을 자동으로 추론할 수 있다.
- 모든 변수에 타입을 직접 쓰지 않아도 된다.

### 타입 명시

- 타입을 직접 적어주는 방식이다.

### 타입 단언

- 컴파일러에게 특정 타입을 강제한 것이다.

### 함수

- TypeScript 함수는 매개변수와 반환값에 타입을 지정할 수 있다.
- 선택적 매개변수는 `?`를 붙여서 표현한다.
- 선택적 매개변수는 보통 뒤에 위치해야 한다.

### interface

- 객체의 모양을 정의하는 설계도 같은 역할을 한다.
- 어떤 속성이 있어야 하는지 규칙을 정할 수 있다.

### Generic

- 타입을 변수처럼 사용할 수 있게 해주는 문법이다.
- 같은 구조를 여러 타입에 재사용할 수 있다.

### tsconfig & strict 모드

- `tsconfig.json`은 TypeScript 컴파일러 설정 파일이다.
- `strict: true`를 설정하면 타입 검사를 더 엄격하게 한다.

## 4. 느낀점

- TypeScript를 사용하면 처음에는 타입을 적는 것이 번거롭게 느껴질 수 있지만, 코드가 커질수록 오류를 미리 잡아주는 장점이 크다고 느꼈다.
- 특히 React에서는 props, state, API 데이터처럼 구조가 중요한 부분이 많기 때문에 TypeScript를 사용하면 유지보수가 훨씬 쉬워질 것 같다.
- `any`를 사용하면 JavaScript와 크게 다를 바가 없어지기 때문에, 가능하면 타입을 명확히 작성하는 습관이 중요하다고 생각했다.
- 앞으로는 단순히 코드가 실행되는지만 보는 것이 아니라, 데이터가 어떤 타입으로 흘러가는지도 함께 확인해야겠다고 느꼈다.
